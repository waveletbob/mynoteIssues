# AI Agent 智能体

## 核心循环

```
感知(理解输入) → 规划(拆解任务) → 行动(调工具/写代码) → 观察(看结果) → 反思 → 循环
```

Agent 不同于普通 LLM 调用：它能**自主决策执行路径**，而不是一问一答。

## 关键能力

| 能力 | 实现方式 | 典型场景 |
|------|---------|----------|
| **工具调用** | Function Calling / MCP | 查数据库、调 API、读写文件 |
| **推理规划** | CoT、ReAct、Plan-Act | 多步骤问题、需要中间推导 |
| **记忆** | 短期(上下文窗口) + 长期(向量库) | 多轮对话、跨会话经验 |
| **自我修正** | 执行→检查→修正循环 | 代码报错后自动修复 |

## 主流框架

| 框架 | 特点 | 适合 |
|------|------|------|
| LangChain Agent | 生态最全，工具多 | 快速原型 |
| LangGraph | 有状态图编排 | 复杂多步流程 |
| CrewAI | 多 Agent 协作 | 角色分工场景 |
| AutoGen (Microsoft) | 对话式多 Agent | 多角色对话 |

## ReAct 模式（当前最实用）

```
Thought: 我需要做 X，先查 Y
Action: 调用 search("Y")
Observation: 返回了 Z
Thought: Z 说明应该用方案 A
Action: 执行方案 A
Observation: 成功
Answer: 最终结果
```

## MCP 与 Agent 的关系

```
Agent (决策层)
  └── 调用工具 → MCP Server (执行层)
                    └── 数据库 / 文件系统 / Kafka / API
```

Agent 负责"想"，MCP 负责"做"。Agent 通过 MCP 获得对外部世界的操作能力。

## 实践要点

- **先用 ReAct 验证可行性**，这个模式覆盖 80% 场景
- **工具设计决定 Agent 质量**：工具描述要精确，参数要约束，输出要结构化
- **设最大步数**：Agent 会无限循环，必须设置 `max_iterations`
- **人工确认高风险操作**：写文件、删数据、执行 shell 需要用户确认
