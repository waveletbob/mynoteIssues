# AI SKILL 技能系统

## 是什么

SKILL 是 AI 的**专业能力包**——不是工具调用（那是 MCP），而是一套"在这个场景下该如何思考和操作的完整指南"。

```
SKILL = 领域知识 + 操作流程 + 最佳实践 + 输出规范
```

## SKILL vs MCP vs Rules

| | SKILL | MCP | Rules |
|------|-------|-----|-------|
| **解决什么** | 如何做好 | 能操作什么 | 不能做什么 |
| **性质** | 方法论 | 工具协议 | 约束护栏 |
| **触发** | 按需激活 | 显式调用 | 始终生效 |

## 技能层级

```
基础技能（代码阅读、文本摘要）
    └── 专项技能（Python 开发、K8s 运维、SQL 优化）
        └── 组合技能（全栈开发 = 前端 + 后端 + 测试）
```

## SKILL 设计要素

| 要素 | 说明 |
|------|------|
| **名称和触发条件** | 让 AI 知道何时激活 |
| **角色定义** | 这个技能的"专家人设" |
| **执行流程** | 第一步做什么，第二步做什么 |
| **注意事项** | 容易踩的坑、必须遵守的规范 |
| **输出规范** | 期望的输出格式和质量标准 |

## 实践示例（OpenSpec 技能）

```
openspec-propose     → 需求提案：分析→设计→生成规格
openspec-apply-change → 实施变更：按任务逐步实现
openspec-verify-change → 验证：对照规格检查实现
openspec-archive-change → 归档：标记完成，更新文档
```

## 实践要点

- **一个 SKILL 只做一件事**，不要写成"万能技能"
- **触发条件要精准**，避免误激活
- **流程要具体**：不说"优化代码"，说"先 profile → 定位热点 → 针对性改写"
- **SKILL 可以组合**：先调用 skill-a 做分析，skill-b 做实现，skill-c 做验证
