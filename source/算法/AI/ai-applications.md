# AI 应用场景

## 编程辅助

| 工具 | 特点 | 场景 |
|------|------|------|
| GitHub Copilot | IDE 内嵌补全 | 日常编码 |
| Cursor | AI-native IDE，全流程 | 项目开发 |
| CodeMaker | Agent 模式，端到端 | 架构设计到部署 |
| Claude Code | 推理能力强，Agent 式 | 复杂重构 |

## 办公自动化

- **文档处理**：AI 写作助手、会议纪要生成、内容摘要
- **数据分析**：自然语言查询数据库、自动可视化、报告生成
- **邮件/沟通**：邮件草拟、信息提取、多语言翻译

## 知识管理（RAG 落地）

- **企业知识库**：内部文档 QA，员工自助查询
- **代码库理解**：用自然语言问项目架构、定位 Bug
- **合规审查**：合同条款检查、政策合规对比

## 客户服务

- **智能客服**：LangChain/LlamaIndex + RAG 构建
- **工单分类**：自动路由 + 优先级判定
- **情绪分析**：对话质量监控

## 专业领域

| 领域 | 典型应用 | 关键能力 |
|------|---------|----------|
| 金融 | 风险评估、研报生成 | 长文档理解 + 数值推理 |
| 医疗 | 病历摘要、辅助诊断 | 领域知识 + 安全合规 |
| 法律 | 合同审查、案例检索 | 长篇对比 + 精确引用 |
| 教育 | 个性化辅导、试题生成 | 知识结构化 + 难度控制 |

## 落地选型建议

```
简单任务 → Prompt API 直接调用
         → 需要领域知识 → RAG
         → 需要多步推理 → Agent
         → 需要外部操作 → Agent + MCP
```

## 实践准则

- **先用 API 验证**，不要一上来就微调模型
- **RAG 的门槛在数据处理**，不在于模型选型
- **Agent 的可靠性靠工具设计**，不靠 prompt 优化
- **安全合规**：医疗、金融领域必须有审核环节
